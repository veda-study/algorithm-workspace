> VEDA 코딩 테스트 스터디는 한화 비전에서 가장 활발히 사용하는 협업 툴인 **Jira**를 기반으로 운영됩니다.
> 단순히 일정을 기록하는 것을 넘어, 매일의 고민과 접근 방식을 기록하여 현업 수준의 애자일(Agile) 프로세스를 경험하는 것을 목표로 합니다.

## 📌 1. Jira 이슈 구조 및 내 카드 찾기

우리는 같은 문제를 각자 풀기 때문에, 번호가 겹치거나 진행 상황이 꼬이는 것을 방지하기 위해 **하위 작업(Sub-task)** 체제로 운영합니다.

1. **상위 작업 (Task):** 스터디 리더가 주차별로 풀 문제를 생성합니다. (예: `[Week 01] 프로그래머스 PCCE 기출문제 1번 문자 출력` ➡️ 이슈 키: `VEDA-1`)
2. **하위 작업 (Sub-task):** 상위 작업 안에 멤버별 개인 카드가 생성됩니다. (예: `명준 - 프로그래머스 PCCE 기출문제 1번 풀이` ➡️ 이슈 키: `VEDA-7`)
3. **주의 사항:** `git commit`을 하거나 데일리 로그를 작성할 때는 **반드시 상위 작업이 아닌 '본인 이름이 적힌 하위 작업의 이슈 키'**(예: VEDA-7)를 사용해야 합니다.

> 💡 **[필독] Jira 이슈 번호(Key)에 대한 안내**
> * Jira는 시스템 구조상 `VEDA-1-1`, `VEDA-1-2` 같은 하위 번호를 지원하지 않고, 어떤 카드든 생성되는 순서대로 프로젝트 전체에서 고유한 번호가 1씩 늘어나며 부여됩니다.
> * 따라서 리더가 문제를 먼저 여러 개 생성해 두면, 상위 문제는 `VEDA-1`인데 내 하위 카드는 `VEDA-7`이 되는 등 **번호가 연속되지 않는 것이 정상**입니다.
> * **핵심 규칙:** 번호가 왜 뛰었는지 신경 쓰지 마세요! 여러분은 오직 jira 칸반 보드에서 **[본인 이름]이 박혀있는 카드 좌측 상단의 이슈 키**만 확인하고, 그 번호로만 커밋과 데일리 로그를 남기시면 됩니다.

## 📊 2. 칸반 보드(Kanban Board) 상태 관리 규칙

매일 공부를 시작하고 마칠 때, 본인 카드의 상태(Column)를 아래 규칙에 맞게 드래그하여 이동시킵니다.

* **TO DO (할 일):** 이번 주에 나에게 할당된 풀지 않은 문제들.
* **IN PROGRESS (진행 중):** 내가 현재 오늘 치열하게 고민하고 코드를 짜고 있는 문제.
* **IN REVIEW (리뷰 대기 🔍):** 문제를 최종 완료하고 GitHub에 PR(Pull Request)을 올려 팀원들의 코드 리뷰를 기다리는 상태.
* **DONE (완료 🎉):** 팀원의 Approve를 받고 `main` 브랜치에 Squash Merge까지 최종 완료된 상태.

## 📝 3. [필수] 데일리 로그(일일 요약본) 작성 규칙

> **"구현을 완료하지 못했더라도, 오늘 어디까지 접근했고 어디서 막혔는지 기록하는 것이 개발자의 핵심 역량입니다."**

문제를 당일에 마스터하지 못했더라도, **매일 공부를 마칠 때(24:00 전까지) 본인의 Jira 하위 작업 카드 내부 `댓글(Comment)` 공간**에 아래 양식으로 데일리 로그를 반드시 작성합니다.

### 📋 데일리 로그 작성 템플릿 (복사해서 댓글에 사용)
```markdown
### 📅 2026.05.18 데일리 진행 상황
* **현재 상태:** ⏳ 진행 중 (💻 로직 구현 중 / 또는 ⚠️ 에러 수정 중 / 또는 🎉 완료)

* **오늘의 접근 방식:**
  * C++ 방식인 `std::cout`과 `std::cin`을 사용해 기본 입출력 틀을 잡음.

* **막힌 부분 및 알게 된 점 (공식 레퍼런스 참고 내용):**
  * `std::endl`은 `'\n'`과 버퍼를 비우는 `flush()`도 호출한다는 것을 알게 되었음.
```

## 🔗 4. GitHub 자동 연동 확인하기

우리가 세팅한 GitHub for Atlassian 앱 덕분에, 여러분이 git-flow.md 규칙에 맞춰 커밋하고 Push하면 Jira에서 다음과 같은 변화가 자동으로 일어납니다.

* **커밋 연동**: 커밋 메시지에 [VEDA-7]를 넣으면, Jira 카드 우측 개발(Development) 세션에 내 커밋 내역이 자동으로 쌓입니다.
* **PR 연동**: GitHub에서 PR을 올리면 Jira 카드가 자동으로 IN PROGRESS ➡️ IN REVIEW 상태로 알아서 이동합니다.
