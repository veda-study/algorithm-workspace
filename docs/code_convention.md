# 📜 C++ 코딩 컨벤션

> VEDA 코딩 테스트 스터디의 코드 가독성과 유지보수성을 높이기 위한 규칙입니다. 
> 실제 현업 및 코딩 테스트 환경에서 실수와 메모리 낭비를 줄이는 것을 목표로 합니다.

## 1. 명명 규칙

기본적으로 단어의 조합은 **카멜 표기법**을 따릅니다.

* **변수 및 함수명:** 소문자로 시작하는 카멜 표기법을 사용합니다.
  * *Example:* `int studentCount;`, `void calculateSum();`
* **클래스 및 구조체명:** 대문자로 시작하는 파스칼 표기법을 사용합니다.
  * *Example:* `class Node;`, `struct Point;`
* **상수(const, macro):** 전체 대문자와 언더바(`_`)를 사용합니다.
  * *Example:* `const int MAX_SIZE = 100;`

---

## 2. 중괄호 및 들여쓰기

코드의 가독성을 위해 **모든 분기문과 반복문은 조건문이 단 한 줄이더라도 반드시 중괄호(`{ }`)로 감싸야 합니다.** (생략 금지)

### 2.1 함수 정의
함수의 시작과 끝 중괄호는 항상 **새로운 줄**에 작성합니다.
```cpp
int main()
{
    // 함수의 시작은 무조건 새 줄에서
    return 0;
}
```
### 2.2 분기문 및 반복문
`if`, `else`, `while`, `for` 등의 제어문은 중괄호를 조건문과 같은 줄에 붙여서 작성합니다.
```cpp
// if - else 문 예시
if (isExist) {
    // 로직 처리
} else {
    // 로직 처리
}

// while 문 예시
while (count < 10) {
    count++;
}
```

## 3. 코딩 테스트 성능 최적화 규칙

시간 초과와 메모리 초과를 방지하기 위해 문제 풀이 소스코드 상단에 아래 규칙을 강제합니다.

### 3.1 Fast I/O 적용
C++의 `cin`, `cout`은 C의 `stdio`와 동기화되어 있어 속도가 느립니다. 대량의 데이터를 입출력할 때 시간 초과를 막기 위해 `main()` 함수 최상단에 항상 아래 세 줄을 포함합니다.
```cpp
int main()
{
    ios_base::sync_with_stdio(false);
    cin.tie(NULL);
    
    // ... 문제 풀이 로직
}
```

## 4. Modern C++ 및 메모리 관리 규칙

임베디드 및 고성능 시스템을 다루는 한화 비전 특성상, 메모리 낭비와 구식 C 스타일 코드를 지양하는 습관을 들여야 합니다.

### 4.1 포인터 초기화는 항상 nullptr
C 스타일의 `NULL`이나 숫자 `0` 대신, C++11 표준인 `nullptr`을 명확하게 사용합니다.
```cpp
int* ptr = nullptr;  // OK
int* oldPtr = NULL;  // 금지
```

### 4.2 대형 오브젝트는 참조자(&)로 전달
std::vector나 std::string 같이 크기가 큰 객체를 함수의 파라미터로 넘길 때는 복사 비용을 줄이기 위해 반드시 상수 참조자(const reference)를 사용합니다.
```cpp
// 메모리 복사가 일어나 비효율적인 방식 (금지)
void printVector(vector<int> v) { ... }

// 복사 없이 메모리 주소만 참조하는 효율적인 방식 (권장)
void printVector(const vector<int>& v) { ... }
```

### 4.3 자료형 선택과 오버플로우 방지 (`long long`)
* 문제 조건에서 연산 결과가 $2 \times 10^9$ (약 20억)을 넘어설 가능성이 있다면, `int` 대신 반드시 `long long` 자료형을 사용합니다.
* 무조건 `int`만 쓰지 말고, 문제의 입력 제한(N의 범위)을 보고 최대 연산 범위를 먼저 계산하는 습관을 들입니다.
